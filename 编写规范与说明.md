总稿分为五大块

- 主讲
- 前端
- 后端
- AI
- 公共

## 命名规范

``` text
part<N>-描述.md
```

这里的part为在总稿中的第n个环节。从下一人之前，上一人之后切割成一个part。
也就是说在各个职责中命名的不是自己流程的part，而是总流程的part。

---

## 公共负责区

公共负责区为内容并非绑定为某一个人身上，可以是主讲可以是前端。
也包含两个人的对话part。
例如在项目展示部分的小游机器人功能展示。
则由前端和AI两个人维护。

---

## 编写规范

#### 公共规范
1. 请勿独立维护公共负责区内的Part（非对话内容除外）。
2. 维护自己的part时，应当考虑下一part的衔接。
3. 在编写PPT翻页点时请严格按照PPT去写。
4. 在维护自己的Part时，如有需修改的PPT页面请创建updatePPT.md在负责区内。
### PPT翻页点

`注意：请遵守公共规范第3条`
PPT翻页点的格式应当为:
```
[PPT切换]
```
如有需添加的PPT，请编写以下格式。
```
[需修改PPT][对应updatePPT.md中的第几条]
```
### updatePPT.md
基本格式
1. 要包含XXX内容。
2. 要删除XXX内容。
3. 要更新XXX内容。
4. ....

# 1. 要包含XXX的内容。

在这里详细描述本页PPT应当编写什么内容（内容为主，排版和样式不需要描述，如果你没有idea的话），怎么修改。
